Fluig Data An end-to-end data exploration platform
A high level overview of Fluig Data. Data pipelines are created from various Data Sources to build Golden records, which serve as a single of truth. Following that, this records are distributed for various purposes.
In Fluig Data, data from multiple sources are mapped to Data Models, like the Customer Data Model shown here. Rules are then set up to protect the resulting Golden records from low quality data.

About Fluig Data

Enterprises need clean data to generate and track business metrics, but many of them tend to have business units that generate and store partial information of the same entity in siloed systems, leading to duplicate and inconsistent information. Fluig Data addresses this by centralizing all information about an entity into a Data Model, then applying rules to reject, duplicate and transform incoming data to produce a reliable single source of truth which is then used to generate insights.

After the records are processed, a dashboard gives an overview of the outcome

User Interface and Prototyping

Building consistent, reusable and flexible user interface patterns help to build users' familiarity with the application. It also makes it easier to build new pages with a more maintainable codebase.

Building a product is an extremely involved process with many levels of UI, flows, states and interactions. Different prototyping tools help communicate details of a variety of flows and interactions to stakeholders.

Framer is generally more suited for mobile prototypes at the moment, but one benefit it provides is to be able to write logic to allow parallel flows within the same prototype.
Prototyping on the web provides several benefits. You benefit from the HTML layout behavior, are able to reuse existing styles from the application, and can be extremely flexible by using Javascript frameworks and libraries like Angular and Greensock (used in this prototype).
Built in React Native, the mobile companion app allows various teams within the company to access key information about customers and other companies

Involvement

As part of a team of about 15, and as one of two UX designers, I have been broadly involved in the development of Fluig Data from the beginning and in various capacities. My contributions have been mostly in the area of designing the user interface, flows and interactions, conducting interviews and usability studies, but I have touched on several other areas as well.

Some highlights:

  • I initiated and conducted the first user research activities in the company, including interviews and usability studies.
  • Before we had a dedicated UI engineer, I built the UI on top of our framework (Angular Material at that time) and later built custom components to better flows specific to our product.
  • After the release of Invision Craft's nested symbols, I re-created the UI components used in the product and centralized them using Invision Library to sync them across multiple documents and make them accessible to the team.
  • I defined a standard for task specifications so that the team can have a shared understanding of the information they needed before they began working on a task.
  • I convinced the team to switch over to Trello from JIRA because I wanted to make it easier and faster to create and get information about development stories.
  • For a significant portion of the development, I conducted sprint planning, retrospectives and prioritized stories. Having visbility over user needs and business goals helped guide many of my decisions.
